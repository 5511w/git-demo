### 进程管理
1. kill [信号类型] 进程PID
---
kill 命令用于关闭一个进程  
其中信号类型有很多种,可以通过kill-I查看所有信号类型。常用的信号类型有SIGKILL,对应的数字 为9,还有SIGTERM和SIGINT,对应的数字分别为15和2。

kill -9 进程PID:表示强制结束进程

kill -2 进程PID:表示结束进程,但是并不是强制性的,常用的ctl+c组合键发出的就是一个 kil-2的信号。

kill -15 进程PID:表示正常结束进程,是kill的缺省选项,也就是ki不加任何信号类型时,默 认类型就是15。

PID是Linux系统中用于区分不同进程的唯一标识，最大为32768，所有的进程都是PID为1的systemd的进程后代

2. kill PID 关闭进程
```kill PID        # 温柔关闭
kill -9 PID     # 强制杀死（最常用）
pkill 进程名     # 不用PID，直接按名字杀
```

3. killall 按进程名批量关闭
killall也用于关闭进程，与kill不同的是，killall后面接进程的名字，而不是进程PID，因此可以关闭一组进程

```
killall 进程名称 #示例killall nginx
killall -9 进行名称 #killall -9 nginx
```

3. rm 删除（高危慎用），CentOS 没有回收站，rm 删除直接永久删除

|选项|作用|说明|
|----|----|----|
|-f|强制删除|忽略不存在的文件、不提示确认、覆盖只读权限|
|i|互交式确认|删除每个文件前都提示，输入 y 确认、n 取消。默认带有-i|
|I|简化互交|删除≥3 个文件 / 递归删除时，只提示一次确认|
|-r|递归删除|删除目录及目录下所有子文件、子目录（删目录必须加）|
|-v|显示详情|输出正在删除的每个文件 / 目录路径|
|-d|删除空目录|只能删空目录，非空会报错|

删除单个文件
---
```rm test.txt       # 普通删除，只读文件会提示
rm -f test.txt    # 强制删除，不提示
```

删除多个文件
---
```
rm file1.txt file2.jpg
rm *.log          # 删除当前目录所有.log文件
```

删除目录（必须加 -r）
---
```
rm -r mydir       # 递归删目录，有提示
rm -rf mydir      # 强制递归删目录（最常用、最危险）
rm -ri mydir      # 安全：删目录前逐项确认
```

* 危险禁区（绝对禁止）
---
```
rm -rf /          # 删除根目录，系统直接崩溃、数据全毁
rm -rf ~          # 删除当前用户家目录，个人数据全没
rm -rf /*         # 同上，删根下所有内容
```

6. useradd / userdel / usermod

### useradd
  
用于新建一个 Linux 用户，示例：useradd test 表示新建一个名为test 的用户

|选项|选项|
|----|----|
|-u uid|指定用户的唯一标识号（UID），该标识号必须全局唯一|
|-g group|指定新建用户登录时的主组（默认组），该组必须已存在|
|-G group|指定新建用户的附加组，该组必须已存在；
当用户同时属于多个组时，登录时的默认组为主组，其余组为附加组|
|-d home|指定新建用户的默认主目录；
若不指定，系统会在 /etc/default/useradd 定义的目录下创建用户主目录|
|-s shell|定新建用户的默认 Shell；
若不指定，系统以 /etc/default/useradd 中定义的 Shell 作为新建用户的默认Shell|

* **主组**：用户登录系统时默认生效的组，一个用户有且只有一个主组
**附加组**：用户额外加入的组，一个用户可以有多个附加组，用于分配额外权限
**shell**：常见默认 Shell 为 /bin/bash，若设置为 /sbin/nologin，则该用户无法登录系统（常用于服务账户）

### userdel

userdel 是删除用户，用于彻底移除系统中的用户账户

|选择|作用|说明|
|----|----|----|
|-r|删除用户 + 家目录 + 邮件池（最常用）|彻底清理，不留残留，生产环境推荐用这个|
|-f|强制删除（即使用户当前登录）|慎用！会强制踢掉在线用户，可能造成数据异常|

* 不加 -r 只会删除 /etc/passwd 里的用户记录，家目录 /home/用户名会残留，容易造成磁盘浪费和权限隐患
* 不能删除当前正在登录的用户，需先 pkill -u 用户名踢下线，再用 -f 强制删除
* 系统用户（UID<1000，如 bin、daemon）禁止随意删除，会导致系统服务崩溃

### usermod

usermod 用于修改已存在用户的各项属性

|选项|作用|对应useradd 参数|说明|
|----|----|----|----|
|-u UID|修改用户的UID（UID必须唯一）|-u|改用户ID，需确保新UID 未被占用|
|-g 组名/GID|修改用户的主组（必须组已存在）|-g|覆盖原主组，不是追加|
|-G 组1,组2|修改用户的附加组（覆盖原附加组）|-G|注意：会清空原有附加组，追加用 -aG|
|-aG 组名|追加附加组（不覆盖原有）|无|给用户加组不丢原有权限|
|-d 新家目录|修改用户的家目录	|-d|需配合-m把旧家目录数据迁移过去|
|-s shell|修改用户的默认 Shell|-s|比如改成 /sbin/nologin禁止登录|
|-c "备注"|修改用户的注释信息|-c|对应 /etc/passwd 第 5 个字段|
|-l 新用户名|修改用户名（重命名）|无|不会自动改家目录，需手动同步|
|-L|锁定用户（禁止登录）|无|等价于 passwd -l，密码前加!|
|-U|解锁用户|无|等价于 passwd -u|

```
useradd [选项] 用户名      # 创建用户

7. groupadd / groupdel / newgrp

### groupadd

groupadd 用于新建一个 Linux 用户组，方便统一管理多个用户的权限，比如给开发组统一分配代码目录权限

|选项|作用|对应 useradd 参数|说明|
|----|----|----|----|
|-g GID|指定组的唯一标识号（GID）|-g|必须全局唯一，不指定则系统自动分配（从 1000 + 开始）|
|-r|创建系统组（GID<1000）|无|用于系统服务账户，普通用户组不建议用|
|-f|强制创建（组已存在也不报错）|无|脚本中常用，避免重复创建报错|

### groupdel

用于删除不再使用的用户组，清理无效组，避免权限混乱

|选项|作用|说明|
|----|----|----|
|-f|强制删除（即使组内有用户）|慎用！会强制移除用户的组关联，可能造成权限异常|

* 如果组是某个用户的主组，直接删除会报错，必须先删除用户或修改用户主组

* 如果组仅作为用户的附加组时，可直接删除，不影响用户登录

### newgrp

临时切换用户的当前有效组，切换后新建的文件 / 目录，所属组会变为新组，退出后恢复原主组

* 仅临时生效，不修改用户的主组 / 附加组配置。切换后需用户属于该组（主组或附加组均可）。执行 exit 即可退出临时组，恢复原状态

```

groupdel [选项] 组名     # 删除组

8. 权限管理：chmod /chown

### chmod 修改权限

chmod用于改变文件或目录的访问权限，该命令包含两种用法，一种是包含字母和操作符表达式的字符设定法，宁外一种是包含数字的数字设定法

Linux 权限分三类，对应 3 组用户：

|用户|权限符号|含义|
|----|----|----|
|u (user)|r w x|文件 / 目录的所有者|
|g (group)|r w x文件 / 目录的所属组|
|o (others)|r w x|其他所有用户|
|a (all)|r w x|所有用户（u+g+o）|

权限的含义：

* r (read)：读权限（文件：查看内容；目录：列出文件）
* w (write)：写权限（文件：修改内容；目录：增删文件）
* x (execute)：执行权限（文件：可运行；目录：可进入）

1. 数字权限法（最常用、考试最爱考）

r=4、w=2、x=1，每组用户的权限是三者之和

|数字|权限|含义|
|----|----|----|
|644|rw-r--r--|文件默认权限（所有者可读写，组和其他人只读）|
|755|rwxr-xr-x|目录 / 脚本常用权限（所有者可读写执行，组和其他人可读执行）|
|700|rwx------|仅所有者可操作，安全级别最高|
|600|rw-------|仅所有者可读写（如配置文件、密钥）|

```
# 给文件设置 644 权限
chmod 644 文件名.txt
 
# 给目录设置 755 权限
chmod 755 目录名
 
# 给文件添加执行权限（755=读写执行+读执行+读执行）
chmod 755 脚本文件.sh
```

2. 字符表示法

u：所有者 
g：所属组 
o：其他用户 
a：所有用户
+：添加权限 
-：移除权限 
=：直接赋值

```

# 给【所有用户】添加执行权限
chmod a+x 脚本.sh
 
# 移除【其他用户】的所有权限
chmod o-rwx 文件名
 
# 给【所属组】添加读权限，移除写权限
chmod g+r-w 文件名
```

9. Vi/Vim 编辑器

vi 是 Linux 下标准的文本编辑程序，也是 Linux 命令行环境下最常用的文本文件可视化编辑器，几乎所有 Linux 发行版都默认预装

|模式|核心功能|切换方式|
|----|----|----|
|命令模式|输入的字符被解释为 vi 命令，实现光标移动、删除、查找、替换、保存退出等操作，是 vi 强大功能的核心|从输入模式按 Esc 键切换；打开文件默认进入此模式|
|文本输入模式|输入的字符作为文件正文内容，用于编辑修改文本|在命令模式下输入 i/a/o 等命令切换|

|模式切换类命令	|功能说明|
|----|----|
|i|在光标当前位置之前插入文字|
|a|在光标当前位置之后插入文字|
|o|在当前行的下方新开一行并进入输入模式|
|O(大写)|在当前行的上方新开一行并进入输入模式|
